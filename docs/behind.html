Behind the Scenes

Layering Code, Memory, and Cake

This page peels back the layers of my senior project for Grandma’s German Chocolate Cake. Instead of just listing what I did, I’m using a visual “cake” and a timeline to show how XML, XSLT, HTML/CSS, and React all stack together.

The Project as a Cake

Each layer of this SVG cake represents a part of the project: XML and schema, XSLT, HTML/CSS layout, and the interactive React site. Click a layer to jump to that part of the story.

Project Goals: Why This Cake?

I chose my Grandma’s German sweet chocolate cake because it has been a long-standing family tradition. For my senior project, I wanted to:

  • Preserve a 2005 handwritten recipe card as a digital heritage object.
  • Show how humanities questions (memory, labor, authorship) connect to code.
  • Build both a readable XML/XSLT-based site and a separate React site based on the same core story.

XML & Schema: Turning the Recipe into Data

The first step was to encode the cake and frosting recipes in XML instead of leaving them as plain text. I modeled:

  • Ingredients, steps, tools, and temperatures with their own elements.
  • People with IDs (for example, njk for my Grandma and eek for me) so I can link them to the people.html page.
  • A Relax NG schema to control what appears where and keep the file valid and reusable.

This XML layer is like the frosting: it holds everything together and gives the project structure before anything is styled or made interactive.

XSLT: Baking the XML into HTML

Once the XML was stable, I wrote XSLT to transform it into the HTML pages that appear on this site. The stylesheet:

  • Turns recipe elements into headings, lists, and paragraphs.
  • Builds links into the people section using the IDs in the XML.
  • Pulls in editorial notes so I can explain older terms or give modern baking advice.

This is the point where the XML becomes something a reader can easily follow, without knowing anything about markup.

HTML & CSS: Designing the Reading Experience

After the transformation, I focused on how the site should feel to actually use. Some choices I made:

  • Using headings and semantic tags so screen readers can navigate the page structure.
  • Writing alt text for images of the handwritten card and baking steps so the visuals are still meaningful without the photos.
  • Reusing containers like the white-box layout to keep pages consistent and readable.

This layer is where the site starts to look like a digital recipe booklet that still respects accessibility and good design.

React Layers Site: Making the Cake Interactive

In addition to this non-React site, I built a React version where the cake itself becomes an interface:

  • The cherry and each cake layer are clickable shapes.
  • Clicking a layer changes the associated text color to match that layer, reinforcing the idea of “reading” the cake visually.
  • The order of the layers (cherry, frosting, top, middle, bottom) matches the way the physical dessert is actually built.

The React site shows how the same core story can be retold as a playful, interactive experience.
